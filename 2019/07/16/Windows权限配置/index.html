Windows权限配置

Posted on 2019-07-16

背景介绍

Win10家庭版取消用户组的权限相关管理功能，所以，只有通过 右键 -> 属性 -> 安全 -> 编辑，几个操作来修改对应文件夹的用户组权限，如下图所示：


如果当前的用户组具备完全控制、修改等足够的权限则能够对当前的各组的权限进行配置，如下图，则权限能够配置成功。

但如果当前的默认组的权限不够大，则会出现访问被拒绝的提示，权限修改将会不成功。如下图将SYSTM的权限只设置为读写，再通过”编辑”选择权限时就会报错。主要原因是当前的用户组只有读写权限，没有权限对所有的用户组权限进行修改。

解决方案

1、当用户组中有处于”完全控制”权限的用户，如下图中的SYSTEM(最高级管理员)。则在属性中选择”高级“，选择”更改”，再选择”高级”,在”立即查找”中选择”SYSTEM”，如下图所示。确定后即可对用户组权限进行更改了

2、如果进行到上一步也报错没有权限，可以尝试以下方法。利用Windows共享用户权限，改变Windows本地权限。

  • 首先，打开属性中共享中的”高级共享“，选择共享此文件，点击权限，并选择”完全控制”，最后点击”确定“完成共享配置。

  • 在windows查询栏输入：\localhost
    即可打开共享文件夹，如下图：

  • 接着通过共享文件夹用户权限，创建一个本地最高权限用户。具体操作如下，打开共享文件夹的属性 -> 安全 -> 添加 -> 选择主体-> 高级 ->”选择一个用户“ -> 确定

  • 经过上一步，用户主体就是上一步所选的用户对象了，并且基本功能可以进行选择了，选择”完全控制“，完成后退出，就建好了一个具有完全控制权限的新用户了。

  • 最后回到C盘属性，到高级安全设置中，进入”审核“，选上”替换所有的子项目“，然后确定，系统就开始自动为所有子文件赋予新的权限了。中间可能会有许多的”进程无法访问“的提示出现，点击”继续“，不用管它即可，等待完成所有安全设置。
